# PANOPTES

A global **military-intelligence & geopolitics** platform built entirely on **public / OSINT data**. Strategic context, operational monitoring, and analytic tooling on a geospatial core — a dark "Tactical Futurism" HUD over a deck.gl map + 3D globe.

> No classified data. No user accounts. Everything is public-source, attributed, and labelled with a confidence tier and freshness state.

## Quick start

```bash
npm install
npm run geo:build      # normalize curated datasets -> public/geo (needs static-data/)
npm run dev            # http://localhost:3000
```

The app runs **without any API keys** — USGS, GDELT, ReliefWeb, adsb.lol, World Bank, CelesTrak, NGA and the RSS feeds need none. Keys only unlock extra realtime feeds (vessels, fires, outages). Copy `.env.example` to `.env.local` to add them (the legacy `src/.env` is also auto-loaded server-side).

## Architecture

- **Next.js (App Router) + React 19 + TypeScript**, Tailwind v4 design tokens from the tactical design system.
- **deck.gl** for all WebGL rendering (`MapView` flat / `GlobeView` 3D), free token-less **CARTO dark** raster basemap.
- **Zustand** for client state (layers, view, selection, timeline); **TanStack Query** for server/async data.
- **Layer registry** (`src/config/layer-registry.ts`) is the single source of truth — one entry per layer drives rendering, the left rail, clustering and health. Rendering/clustering/filtering stay generic over a normalized `GeoEntity`.
- **Cached serverless proxies** (`src/app/api/*`) wrap every upstream with an in-memory TTL cache, a **circuit breaker**, and **serve-stale-on-error**; real status is reported via the `X-Panoptes-Health` header (live / degraded / stale / offline).

### Data tiers

- **Tier A — bundled GeoJSON** (`public/geo/*`, generated by `scripts/build-geo.mjs`): borders, military bases, ports, cables, data centers, dams, minerals, launch sites, water conflicts.
- **Tier B — PMTiles vector tiles** (`scripts/build-tiles.mjs` → `public/tiles/*`, served from object storage): roads, oil & gas pipelines/wells, reservoirs. Streamed on-demand via a deck.gl MVT layer.
- **Tier C — live serverless proxies**: earthquakes, fires, disasters, conflict (media), flights, vessels, satellites, outages, news.
- **Tier D — country indicators** (`public/data/country-indicators.json`): joined to borders for choropleths.

## Data dump

The raw source datasets (~130 GB) are **not** in git. The small derived runtime assets (`public/geo`, `public/data`) **are** committed so the app deploys standalone. To regenerate from source, restore the dump into `static-data/` and run `npm run geo:build` / `npm run geo:tiles`.

## Deployment

Deploys to **Vercel** (static + serverless). Add API keys as project env vars. Upload `public/tiles/*.pmtiles` to free object storage (Cloudflare R2 / S3 / Vercel Blob) and point layers at those URLs. `.github/workflows/data-refresh.yml` rebuilds derived assets on a schedule when a `DATA_DUMP_URL` secret is provided.

## Licensing & attribution

Each layer carries its source + license in the registry and is shown in-app. Notable: Natural Earth / USGS / NGA (public domain), adsb.lol & GRIP (ODbL), TeleGeography, Pacific Institute, OGIM/EDF, World Bank, CelesTrak, ReliefWeb (OCHA). Reference apps (World Monitor, Pharos, WorldWideView) were studied for **patterns and upstream APIs only** — no AGPL/Elastic-licensed code was copied. Nuclear/ballistics models are re-implemented from public scaling laws (Glasstone & Dolan), not from NUKEMAP/MISSILEMAP source.

## Notable deviations from the plan

- **Next 15.5** (stable) rather than 16, with React 19 — identical App-Router architecture, lower toolchain risk.
- **deck.gl-only basemap** (CARTO dark raster + deck MVT for PMTiles) instead of MapLibre + Protomaps, giving one unified flat/globe renderer with no token. MapLibre/Protomaps remains a drop-in alternative (`NEXT_PUBLIC_BASEMAP_PMTILES_URL`).
- **Satellites via CelesTrak** (redistribution-permitted) instead of Space-Track, avoiding the bulk-TLE ToS issue while still propagating with `satellite.js` in a Web Worker.
